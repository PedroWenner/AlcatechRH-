# PROJECT STRUCTURE

## Purpose

Definir estrutura de pastas e arquivos para projeto React + Tailwind.

---

## Principles

- Separar components, pages, layouts, hooks e services  
- Componentes reutilizáveis em components/  
- Páginas em pages/  
- Layouts em layouts/  
- Hooks em hooks/  
- Integração com APIs em services/  

---

## Guidelines

- Cada componente deve ter seu próprio arquivo  
- Cada página deve ter um arquivo principal  
- Pastas devem refletir responsabilidades, não tipo de arquivo

# BUG REPORTING

## Purpose

Padronizar o **relato de bugs**, facilitando correção e rastreabilidade.

---

## Principles

- Relatar bugs com passos claros para reprodução  
- Incluir informações relevantes: ambiente, versão, dados de entrada  
- Priorizar severidade e impacto  

---

## Guidelines

- Usar ferramentas de tracking (Jira, Trello, GitHub Issues)  
- Anexar logs, screenshots ou gravações sempre que possível  
- Manter histórico de bugs reabertos ou pendentes

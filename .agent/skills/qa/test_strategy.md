# TEST STRATEGY

## Purpose

Definir a **estratégia de testes** para garantir cobertura adequada e qualidade do software.

---

## Principles

- Priorizar funcionalidades críticas e fluxos principais  
- Balancear **testes manuais e automatizados**  
- Reusar casos de teste quando possível  
- Manter rastreabilidade entre requisitos e testes  

---

## Guidelines

- Criar plano de testes para cada release  
- Definir critérios de sucesso e falha  
- Revisar estratégia periodicamente

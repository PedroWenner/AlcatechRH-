# ACCESSIBILITY

## Purpose

Garantir que a aplicação seja acessível para todos os usuários.

---

## Principles

- Seguir guidelines WCAG  
- Usar **roles e ARIA labels** corretamente  
- Evitar cores que dificultem leitura  

---

## Guidelines

- Inputs devem ter labels associados  
- Componentes interativos devem ser navegáveis via teclado  
- Feedbacks visuais devem ter suporte a leitores de tela

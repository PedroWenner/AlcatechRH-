# DOCUMENTATION

## Purpose
Documentar todas as solicitações de usuários seguindo template padronizado.

## Guidelines
- Criar arquivo na pasta correta:
  - **features** → nova funcionalidade  
  - **bugfixes** → correção de bug  
  - **hotfixes** → correção urgente  
- Incluir:
  - História do usuário  
  - Critérios de aceitação  
  - Prioridade  
  - Notas adicionais
